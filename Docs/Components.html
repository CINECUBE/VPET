VPET Componenets

VPET consists of 3 major applications: the Unity based client, a scene distribution plugin and a synchronisation server

Unity Client Application

This is the main application that allows interaction on tablets and head mounted devices. It is currently built to interact with Katana/Live View but could be adapted to any other central development system. Scene objects, lights and animation can now be edited through an simple and intuitive interface. Latest hardware like Google Tango support positional tracking for exploration of the virtual scene in space. When building your Unity client, make sure you use the appropriate Unity version. Find more information in the detailed documentation.

As of spring 2018 we provide a precompiled VPET client via the Apple App Store.

Scene Distribution Plugin

The scene distribution plugin traverses the Katana/Live View scene and communicates its scene structure, textures, meshes and lights to the VPET Unity client. The scene will be build from scratch each time a client connects. This removes the need for a dedicated export procedure. Scenes might hold a optimized version for mobile devices with low resolution versions of the objects.

Synchronization Server

This server synchronizes all updates to the scene either by mobile VPET clients or the central production system (Katana/Live View). This includes updates from tracking devices like the ncam reality system.
